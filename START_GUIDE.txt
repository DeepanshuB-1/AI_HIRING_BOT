--------------------------------------------------------------------------------
PREREQUISITES (one-time checks)
--------------------------------------------------------------------------------

1. PostgreSQL is running on port 5433
   - Check: open pgAdmin or run:  pg_isready -p 5433
   - Database name: hiringbot
   - The pgvector extension is auto-created by the app on first run

2. Redis is running on port 6379
   - Check: redis-cli ping   (should print PONG)
   - Start if not running:  redis-server   (or via Windows Services)

3. Ollama is running with the required models pulled
   - Check: http://localhost:11434
   - Pull models (only needed once):
       ollama pull qwen2.5:7b          ← analysis/scoring/reports (better JSON accuracy)
       ollama pull llama3.2:3b         ← voice interview (faster responses)
       ollama pull nomic-embed-text    ← embeddings (required)

   Optional upgrades (need 10GB+ VRAM):
       ollama pull qwen2.5:14b         ← better analysis (set OLLAMA_ANALYSIS_MODEL=qwen2.5:14b in .env)

   Legacy (if you pulled these before, they still work):
       ollama pull llama3.1:8b
       ollama pull mistral:7b

   - Start Ollama app or run:  ollama serve

4. .env file exists at project root with real keys
   - Do NOT commit this file — it is gitignored
   - Key variables:
       DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5433/hiringbot
       REDIS_URL=redis://localhost:6379
       SECRET_KEY=<your secret>
       SENDGRID_API_KEY=<your key>
       TWILIO_ACCOUNT_SID=<your sid>
       TWILIO_AUTH_TOKEN=<your token>
       TWILIO_PHONE_NUMBER=<your twilio number>
       ELEVENLABS_API_KEY=<your key>
       ELEVENLABS_VOICE_ID=<your voice id>
       DEEPGRAM_API_KEY=<your key>
       WEBHOOK_BASE_URL=<your ngrok url>  ← update this each session (see step 6)

STEP 6 — Start ngrok (only needed for voice calls)  [Terminal 5]
================================================================================

Then RESTART the FastAPI server (Ctrl+C in Terminal 1, run uvicorn again)
so it picks up the new URL before placing any calls.

The ngrok URL changes every session unless you have a paid ngrok account.

================================================================================
COMMON ISSUES
================================================================================

"Cannot connect to database"
  → PostgreSQL is not running, or DATABASE_URL in .env is wrong (check port 5433)

"Redis: UNREACHABLE" in FastAPI logs
  → Redis is not running. Run: redis-server

"Task stuck / pipeline not running"
  → Celery analysis worker is not started (Terminal 2 missing)

"Emails not sending"
  → Celery notification worker not started (Terminal 3 missing)
  → SENDGRID_API_KEY missing/wrong in .env

"Voice call not connecting / webhook errors"
  → ngrok is not running, or WEBHOOK_BASE_URL in .env is stale
  → Restart FastAPI after updating the URL

"Ollama model not found" error
  → Run: ollama pull llama3.1:8b  (and mistral:7b, nomic-embed-text)

venv not found
  → Run from project root: python -m venv venv
  → Then: pip install -r requirements.txt

"First voice call is very slow to transcribe"
  → faster-whisper downloads the Whisper medium model (~769MB) on the very first call
  → Subsequent calls are fast (model stays loaded in memory)
  → To pre-download: python -c "from faster_whisper import WhisperModel; WhisperModel('medium')"
